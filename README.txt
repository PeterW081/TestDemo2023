　　《專案介紹》

　　該個專案為面試作品。專案成品為一組 C/S 架構的網絡服務程式（包括 Server 和 Client），通過 Grpc 控制遠端 RaspberryPi 上的 Led 和 WaterLevel。
　　該個專案使用 C++ 編寫，使用 Cmake 構建。該個專案演示內容包括 GRpc、PiGpio、GTest、QtWidgetGui。該個專案作為面試作品，程式代碼有所取捨，儘量精簡直觀。

　　程式網絡通訊部分，使用 GRpc 實現功能。使用 GRpc 直接包裝 PiGpio，並再進一步包裝出一套「遮蔽網絡訪問操作」的 Api。如此利多在於，簡化遠程設備控制的開發難度，並且提高代碼可讀性。該個 Api 的下游開發者，可以如同調用本端設備一般地調用遠端設備，並且不用耗費太多精力在網絡通訊方面。
　　程式 Server 部分，使用 GRpc C++ 作為框架，使用 PiGpio 控制單晶片機器上的 GPIO 設備。Server 乃是運行於 Linux 之下的程式，而非直接運行於單晶片機器上的裸機程式。
　　程式 Client 部分，使用 QtWidget 作為框架。使用自行包裝的 Api，控制遠端單晶片機器上的 GPIO 設備。

　　該個專案開發過程，還有涉及 C++ 的交叉編譯和 Linux 下的 Systemd，此中為了避免主題失焦不作繁述。
　　鑒於個人術業專攻。ClientGui 視覺呈現方面儘量從簡設計。RaspberryPi 硬體部分乃是雜貨鋪裡購買的現成品，硬體操作僅是 GPIO 的簡單操作。
　　最後提下，若要使用 GRpc 串物聯網，此中並非理想設計。如果實有這方面需求，可於面試時候討論。
